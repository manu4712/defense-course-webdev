Learn the fundamentals of computer networks

Day 1 — Computer Networks (CN)

Understanding how the internet works from first principles

The internet is not magic—it's a series of logical solutions to fundamental problems of sharing information between computers across the world.

How the Internet Works

The internet evolved through solving five core problems, each building on the previous solution.

Principle 1: The Need to Share Information

The fundamental challenge is moving information from Computer A to Computer B when they're not in the same location.

The Basic Solution: Physical connection via cables (ethernet, fiber optic) carrying electrical signals or light pulses. We establish a simple code: high voltage = "1", low voltage = "0".

Result: The most basic network—a direct link sending bits between two computers.

Principle 2: The Problem of Scale

Direct connections fail with multiple computers. Three computers need 3 cables, four computers need 6 cables, 100 computers would need 4,950 cables.

The Solution: Central device (Switch) where every computer connects to one point. When Computer A wants to talk to Computer C, it sends the message to the switch, which forwards it only to Computer C.

Result: Local Area Network (LAN)—computers in close proximity communicate efficiently.

Principle 3: Connecting Networks

Local networks need to communicate with other local networks across distances.

The Solution: Special computers called Routers that connect to multiple networks and pass messages between them. These devices "route" traffic from one network to another.

Result: The Internet—a "network of networks" connecting millions of private and public local networks through routing devices.

Principle 4: Finding Anyone (Addressing)

In a global web of networks, we need unique addresses for every device to ensure messages reach their destination.

The Solution: IP Address system—unique labels (like 142.250.184.142) assigned to each device. Routers examine destination IP addresses and forward messages to the next best "hop" toward the final destination.

Result: Universal addressing system enabling global device identification and message routing.

Principle 5: Reliability and Fairness

Large files sent as single streams create problems: they monopolize connections and become corrupted easily.

The Solution: Break large files into small, numbered packets containing:

  • Small piece of data (payload)
  • Destination IP address
  • Sender's IP address
  • Sequence number for reassembly

Result: TCP/IP protocol system for reliable, fair data transmission through packet switching.

IP Addressing Systems

IPv4 Addresses

Structure: 32-bit number divided into four 8-bit sections, written as decimal numbers 0-255.

Example: 172.217.16.142 (Google's address)

Limitation: Only 4.3 billion possible addresses (2^32), leading to address scarcity with modern device proliferation.

Public vs Private IP Addresses

Public IP Address:

  • Globally unique address on the internet
  • Assigned by Internet Service Provider (ISP)
  • Used by servers and routers for internet-facing services
  • Example: Your home router's external address

Private IP Address:

  • Local address within private networks only
  • Must be unique only within the local network
  • Assigned by local router
  • Reserved ranges:
    • 10.0.0.0 to 10.255.255.255 (large corporations)
    • 172.16.0.0 to 172.31.255.255 (medium networks)
    • 192.168.0.0 to 192.168.255.255 (home networks)

IPv6 Addresses

  • Purpose: Solve IPv4 address exhaustion
  • Structure: 128-bit address providing 340 undecillion possible addresses (2^128)
  • Format: Eight groups of four hexadecimal characters separated by colons

Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

MAC Addresses

  • Definition: Unique, permanent serial number burned into network hardware
  • Structure: 48-bit number displayed as 12 hexadecimal digits
  • Common formats:
    • Colon-separated: 3C:22:FB:A3:B4:C5
    • Hyphen-separated: 3C-22-FB-A3-B4-C5
    • Period-separated: 3c22.fba3.b4c5
  • Finding MAC Address:
    • Windows: ipconfig /all or getmac /v
    • macOS: ifconfig

Port Numbers

Definition: Numerical identifiers (0-65535) that direct network traffic to specific applications or services on a device.

Structure: 16-bit unsigned integer providing 65,536 possible ports

Port Categories

Well-Known Ports (0-1023):

  • Reserved for standard internet services
  • Require administrator privileges
  • Key examples:
    • 21: FTP
    • 22: SSH
    • 80: HTTP
    • 443: HTTPS
    • 53: DNS

Registered Ports (1024-49151):

  • For specific applications and services
  • No special privileges required
  • Developer examples:
    • 3000: React/Node.js development
    • 3306: MySQL
    • 27017: MongoDB
    • 8080: Alternative web server

Dynamic Ports (49152-65535):

  • Temporary, automatic use by client connections
  • Not configured manually
  • Assigned by operating system for outgoing connections

Domain Name System (DNS)

Purpose: Translate human-friendly domain names into computer-friendly IP addresses.

DNS Resolution Process

Step 1: Local Cache Check

  • Browser cache (recent visits)
  • Operating system cache
  • Router cache
  • If found, process stops here

Step 2: Recursive Resolver

  • Query sent to ISP's DNS server or public resolver (8.8.8.8, 1.1.1.1)
  • Recursive resolver promises to find the complete answer

Step 3: Hierarchical Search

  • Root Servers: Direct to appropriate TLD servers
  • TLD Servers: Direct to domain's authoritative servers
  • Authoritative Servers: Provide final IP address

Step 4: Return Journey

  • Answer cached at each level with TTL (Time-To-Live)
  • IP address returned to requesting computer

Step 5: Final Connection

  • Browser connects directly to IP address
  • DNS process complete

Key Terms

  • Network: Connected computers sharing information
  • Router: Device connecting different networks
  • Switch: Central device connecting computers in local network
  • Packet: Small, numbered piece of larger data transmission
  • TCP/IP: Protocol system for reliable internet communication
  • ISP: Internet Service Provider supplying internet access
  • TLD: Top-Level Domain (like .com, .org)
  • TTL: Time-To-Live, how long DNS records are cached
  • Recursive Resolver: DNS server that finds complete answers
  • Authoritative Server: Official source for domain's DNS records
