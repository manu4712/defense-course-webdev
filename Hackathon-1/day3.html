Deep dive into nested lists and HTML tables

1. Nested Lists

Nested lists allow you to create multi-level hierarchies. You can nest <ul> or <ol> inside list items.

Example: Company Departments

  • Engineering
    • Frontend Team
      • React Developers
      • UI/UX Designers
    • Backend Team
      • Node.js Developers
      • Database Administrators
  • Marketing
    1. Content
    2. SEO
  • Sales

Best Practices

  • Keep nesting depth reasonable (2–3 levels max).
  • Use semantic nesting to reflect real hierarchy.
  • Ensure proper indentation in code for readability.

2. HTML Tables

Tables organize tabular data with rows and columns. Use <table>, <thead>, <tbody>, <tr>, <th>, and <td>.

Basic Table Structure

<table>
  <thead>
    <tr>
      <th>Header 1</th>
      <th>Header 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Cell 1</td>
      <td>Cell 2</td>
    </tr>
  </tbody>
</table>

Advanced Table Features

You can merge cells across rows or columns using rowspan and colspan.

<tr>
  <td rowspan="2">Group A</td>
  <td>Item 1</td>
</tr>
<tr>
  <td>Item 2</td>
</tr>

Example: Merged Cells

Group Item Quantity
Fruits Apples 10
Oranges 8

Key Terms

  • Nested List: A list within another list for hierarchical data.
  • Table: HTML structure for rows and columns of data.
  • <thead> / <tbody>: Separate header and body sections.
  • <th>: Table header cell.
  • <td>: Table data cell.
  • rowspan: Merge cell vertically.
  • colspan: Merge cell horizontally.

Revision Checklist
