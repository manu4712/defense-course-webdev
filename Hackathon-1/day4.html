Understanding file paths, boilerplate, and containers

1. File Path System

Step 1: The Fundamental Truth

A website is not one single file but a collection of HTML, CSS, JavaScript, images, videos, and fonts organized into folders.

Step 2: The Core Problem

How does index.html tell the browser where to find logo.png? The browser needs an exact address: a file path.

A) Relative File Paths

my-website/
├── index.html
├── about.html
├── images/
│   ├── logo.png
│   └── hero.jpg
└── pages/
    ├── contact.html
    └── terms.html
Scenario 1: Same Folder

<!-- inside index.html -->
<a href="about.html">About Us</a>
Scenario 2: Sub-Folder (Down)

<!-- inside index.html -->
<img src="images/logo.png" alt="My Website Logo">
Scenario 3: Parent Folder (Up)

<!-- inside pages/contact.html -->
<img src="../images/logo.png" alt="My Website Logo">

B) Absolute File Paths

Full URLs starting with http:// or https:// for resources not on your own site.


<!-- external link -->
<a href="https://www.google.com">Search</a>

<!-- external image -->
<img src="https://upload.wikimedia.org/…/html5_logo.svg" alt="HTML5 Logo">

Security “Why”

Browsers sandbox code. Allowing local absolute paths would expose private files. Relative paths work anywhere.

2. HTML Boilerplate

Boilerplate gives the browser essential setup information before rendering content.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>
Why Each Line Matters
  • <!DOCTYPE html>: Triggers standards mode.
  • <html lang="en">: Declares page language for SEO and accessibility.
  • <meta charset="UTF-8">: Ensures proper character encoding.
  • <meta name="viewport">: Enables responsive mobile rendering.
  • <title>: Sets the tab title and bookmark name.

3. Containers & Labels

<div> — The Generic Box

A meaningless container to group elements into a single box you can style.

<div>
    <img src="avatar.png">
    <h2>User Name</h2>
    <p>Bio text.</p>
</div>

class & id — Labels for Boxes

id targets one unique element; class groups multiple elements.

<div id="header">…</div>
<div class="card">…</div>
<div class="card">…</div>

<span> — The Inline Highlighter

An inline container for styling text without breaking flow.

<p>This is <span class="highlight">important</span> text.</p>

4. Semantic Structure Tags

Use semantic containers for meaningful sections:

  • <header>: Intro or navigation area.
  • <main>: Unique page content (one per page).
  • <footer>: Closing content (copyright, links).

Revision Checklist

  • Can you write and distinguish relative vs absolute paths?
  • Do you understand each part of the HTML5 boilerplate?
  • Can you group elements with div and target them with class/id?
  • Are you able to highlight inline text with span?
  • Do you know when to use header, main, and footer?
