Building interactive, data-collecting form interfaces

1. The Fundamental Truth

Websites must collect user input to enable login, search, purchases, comments, and more. Without forms, the web is read-only.

2. The Core HTML Form

The <form> element packages fields and sends data to a server. Use action and method attributes when connecting to backend logic.

The Basic Text Input

<label for="firstName">First Name:</label>
<input type="text" id="firstName" name="firstName">

Connecting Labels

Use id on inputs and matching for on <label> for clickable, accessible form fields.

3. Form Submission

<form action="/submit" method="post">
  …fields…
  <button type="submit">Submit</button>
</form>

The submit button sends data; wrap fields in <form> for one cohesive submission.

4. Field Types & Grouping

Radio Buttons (Single Choice)

<label>T-Shirt Size:</label><br>
<input type="radio" id="sizeS" name="shirtSize" value="small">
<label for="sizeS">Small</label><br>
<input type="radio" id="sizeM" name="shirtSize" value="medium">
<label for="sizeM">Medium</label><br>
<input type="radio" id="sizeL" name="shirtSize" value="large">
<label for="sizeL">Large</label>

Checkboxes (Multiple Choice)

<label>Pizza Toppings:</label><br>
<input type="checkbox" id="toppingPep" name="toppings" value="pepperoni">
<label for="toppingPep">Pepperoni</label><br>
<input type="checkbox" id="toppingMush" name="toppings" value="mushrooms">
<label for="toppingMush">Mushrooms</label><br>
<input type="checkbox" id="toppingOni" name="toppings" value="onions">
<label for="toppingOni">Onions</label>

Textarea (Multi-Line Text)

<label for="comments">Any special instructions?</label><br>
<textarea id="comments" name="comments" rows="4" cols="50"></textarea>

Dropdown Menu

<label for="country">Country:</label><br>
<select id="country" name="country">
  <option value="">--Choose--</option>
  <option value="in">India</option>
  <option value="us">USA</option>
  <option value="uk">United Kingdom</option>
  <option value="au">Australia</option>
</select>

5. HTML5 Enhanced Inputs

Password Field

<label for="userPass">Password:</label><br>
<input type="password" id="userPass" name="userPassword">

Number Field with Range

<label for="userAge">Age (18–99):</label><br>
<input type="number" id="userAge" name="age" min="18" max="99">

Date Picker

<label for="birthDate">Date of Birth:</label><br>
<input type="date" id="birthDate" name="dob">

Key Terms

  • <form>: Container for user-input fields.
  • <label>: Describes and links to form controls.
  • name: Data key sent to the server.
  • value: Data value submitted for inputs.
  • <input>: Single-line, various types (text, radio, etc.).
  • <textarea>: Multi-line text input.
  • <select>: Dropdown for single-choice lists.
  • <button>: Container button, supports HTML content.

Revision Checklist

  • Can you connect label and input with for/id?
  • Do you understand grouping rules for radio and checkbox?
  • Can you differentiate name vs id roles?
  • Are you able to use HTML5 input types (date, number, password)?
  • Can you build a complete form with submission?
