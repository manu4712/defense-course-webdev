Embedding videos, audio, and special characters

1. The Fundamental Truth

The web needs to deliver any content—not just text and images. HTML5 introduced native media tags to replace insecure plugins like Flash.

2. Basic Video Embedding

The <video> tag embeds video files directly into webpages.

Simple Video with Controls

<video src="my-awesome-video.mp4" controls></video>

Video with Dimensions and Poster

<video src="video.mp4" width="640" height="360" 
       poster="images/video-thumbnail.jpg" controls>
</video>

Example Video Player

3. Audio Embedding

The <audio> tag works identically to video but without visual components.

Basic Audio Player

<p>Listen to our theme song:</p>
<audio src="audio/theme-song.mp3" controls></audio>

4. Advanced Video Features

Autoplay (Muted) and Loop

<!-- This will autoplay, but with no sound -->
<video src="video.mp4" autoplay muted loop controls></video>

Multiple Source Formats (Professional Method)

<video controls poster="images/thumbnail.jpg">
  <source src="videos/my-video.webm" type="video/webm">
  <source src="videos/my-video.mp4" type="video/mp4">
  Sorry, your browser doesn't support embedded videos.
</video>

5. Accessibility with Subtitles

Use <track> elements to add captions, subtitles, and descriptions.

<video controls>
  <source src="video.webm" type="video/webm">
  <source src="video.mp4" type="video/mp4">

  <!-- English Captions for hearing impaired -->
  <track src="captions_en.vtt" kind="captions" 
         srclang="en" label="English">

  <!-- Spanish Subtitles for translation -->
  <track src="subtitles_es.vtt" kind="subtitles" 
         srclang="es" label="Español">
</video>

Track Kinds

  • captions: Direct transcription for hearing-impaired users
  • subtitles: Translation into another language
  • descriptions: Audio description for visually impaired users

6. Format Compatibility Comparison

Method Browser Compatibility Flexibility Simplicity Best Use Case
<source> Method Excellent (multiple formats) High More complex Public websites
src Method Good (single format) Low Very simple Internal projects

7. HTML Entities for Special Characters

HTML entities display special characters that browsers might interpret as code.

Copyright Symbol Methods

<!-- Method 1: Entity Name (Recommended) -->
<p>Copyright &copy; 2024 My Website</p>

<!-- Method 2: Entity Number -->
<p>Copyright &#169; 2024 My Website</p>

Common HTML Entities

Symbol Description Entity Name Entity Number Example
© Copyright &copy; &#169; © 2024
® Registered Trademark &reg; &#174; Brand®
™ Trademark &trade; &#8482; Product™
< Less-than &lt; &#60; <p> tag
> Greater-than &gt; &#62; </p> tag
& Ampersand &amp; &#38; A & B
" Double Quote &quot; &#34; He said "Hello"
€ Euro Sign &euro; &#8364; Price: 50€

Why Use Entities?

Browsers interpret < and > as HTML tags. To display literal code examples, use &lt; and &gt;.

Entity Examples in Action

HTML code example: <p>This is a paragraph</p>

Math expression: 5 < 10 & 10 > 3

Key Terms

  • <video>: Embeds video files with native browser controls
  • <audio>: Embeds audio files with playback controls
  • controls: Enables play/pause/volume/timeline controls
  • <source>: Provides multiple format options for compatibility
  • <track>: Adds subtitles, captions, or descriptions
  • poster: Preview image displayed before video plays
  • HTML Entities: Special codes for symbols (&copy;, &lt;, etc.)
  • autoplay/muted: Auto-starts video without sound

Revision Checklist

  • Can you embed video with controls and poster images?
  • Do you understand the difference between single-source and multi-source methods?
  • Are you able to add audio players to your pages?
  • Can you use track elements for accessibility?
  • Do you know common HTML entities for special characters?
  • Can you display literal HTML code using entities?
